PRODUCT CONTEXT

Nexus is an AI product-building operating system.
In simple terms:
A user gives Nexus an idea for a product or business.
Nexus helps the user clarify the idea, understand the audience and problem, break the work into tasks, execute the next task, show the result, ask for confirmation, update project state, and move to the next task.
Nexus is not a dashboard-heavy admin tool.
Nexus is a guided working loop.
The product should feel like a clear AI-powered flow for building and advancing a product, one meaningful step at a time.

SCREEN DEFINITIONS

For each screen, design a full screen and include:
- screen title
- purpose
- what the user sees first
- main content
- primary CTA
- secondary CTA
- what happens after clicking
- empty state
- loading state
- success state
- error state

Use the following product definitions:

1. Home / Project Switcher
- Title: הבית שלך / Home
- Purpose: let the user choose an existing project or start a new one
- User sees first: list of projects and one obvious “create new project” action
- Main content: project cards/rows with status, last updated, active badge
- Primary CTA: Create New Project
- Secondary CTA: Open Project
- After click: either open Project Create or enter the selected project’s loop
- Empty state: no projects yet, invite creation
- Loading state: loading projects
- Success state: project selected successfully
- Error state: unable to load project list

2. Project Create
- Title: מה אתה רוצה לבנות?
- Purpose: capture the initial idea and optional supporting assets
- User sees first: a large clean creation form
- Main content: project name, idea description, file upload, optional URL, helper cards
- Primary CTA: Create Project and Start
- Secondary CTA: none or Save Draft if useful
- After click: create project and enter Smart Onboarding
- Empty state: blank form with guided placeholders
- Loading state: creating project
- Success state: project created, onboarding starts
- Error state: validation or creation failed

3. Smart Onboarding
- Title: רוצה להבין את הפרויקט שלך
- Purpose: AI conversation that clarifies audience, problem, current workflow, success criteria
- User sees first: conversational thread with AI messages and user responses
- Main content: chat, current question, input field, “what I understood” and “what is missing” summary cards
- Primary CTA: Continue
- Secondary CTA: Restart or Back
- After click: next AI question or move to Understanding Summary when complete
- Empty state: first AI question waiting
- Loading state: AI thinking / generating next question
- Success state: onboarding complete
- Error state: AI response failed or session lost

4. Understanding Summary
- Title: זה מה שהבנתי
- Purpose: show the condensed structured understanding before entering the working loop
- User sees first: four summary cards
- Main content: audience, problem, solution, goal
- Primary CTA: Correct, this is right / Continue
- Secondary CTA: Fix / Edit
- After click: either go back to onboarding or continue to Loop Core
- Empty state: no summary available yet
- Loading state: building summary
- Success state: understanding confirmed
- Error state: failed to summarize

5. Nexus Loop Core
- Title: מה עושים עכשיו
- Purpose: show the single most important next task
- User sees first: one big mission card
- Main content: current task, why it matters, what will happen, vertical loop timeline
- Primary CTA: Execute Task
- Secondary CTA: View Proof
- After click: Execute Task opens Execution Live, View Proof opens latest proof
- Empty state: no next task selected yet
- Loading state: selecting next task
- Success state: task is ready to execute
- Error state: failed to resolve next task

6. Execution Live
- Title: מבצעים את המשימה
- Purpose: show the live execution of the current task
- User sees first: active mission card and execution status panels
- Main content: execution steps, current active step, live actions, recent logs
- Primary CTA: Show Proof When Ready
- Secondary CTA: Stop Execution
- After click: primary goes to Result / Proof, secondary stops or pauses execution
- Empty state: no active execution yet
- Loading state: system is executing
- Success state: task execution finished
- Error state: execution failed or partially failed

7. Result / Proof
- Title: המשימה הושלמה
- Purpose: show what Nexus created and why it counts as proof
- User sees first: preview of the created output
- Main content: result preview, proof bullets, metrics/statistics, artifact actions
- Primary CTA: Show Full Proof
- Secondary CTA: Open in New Tab, Download Files
- After click: primary can expand detailed proof or go toward confirmation
- Empty state: no proof yet
- Loading state: assembling proof
- Success state: proof ready
- Error state: proof unavailable or incomplete

8. Confirmation
- Title: מה דעתך על התוצאה?

- User sees first: two large decision cards
- Main content: approve / needs fix decision

- Success state: decision recorded
- Error state: failed to save decision

9. State Update

UX RULES

Every screen must clearly answer:
- what is this?
- why does it matter?
- what should I do now?

Additional UX rules:
- always present one obvious primary action
- secondary actions must support the same goal, not distract from it
- never force the user to decode product state from abstract metrics
- keep the user moving through one clear loop
- explain system reasoning in plain language
- make the AI feel useful, not magical or vague
- use supporting cards only when they help the next decision
- preserve consistency across all screens
- prioritize clarity over decorative complexity

ANTI-RULES

Do not create:
- dashboard clutter
- random panel farms
- inconsistent spacing or typography
- fake buttons
- placeholder-only cards
- generic SaaS admin templates
- unrelated product features
- crowded KPI walls as the main UX
- dark mode unless explicitly needed
- multiple competing CTAs on the same screen
- screens that do not connect logically to the next screen

- clearly rooted in the current Create screen visual style
